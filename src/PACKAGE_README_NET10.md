These package ids (**Serilog.Sinks.SentrySDK.10** and **Serilog.Sinks.SentrySDK.AspNetCore.10**) ship **net10.0-only** assemblies (`lib/net10.0/`). They match the main sink and ASP.NET Core integration, but under alternate ids for apps that target **.NET 10** only.

For **net6.0 and net10.0** in one package, use **Serilog.Sinks.SentrySDK** and **Serilog.Sinks.SentrySDK.AspNetCore**. For **net6.0-only** ids, use **Serilog.Sinks.SentrySDK.6** / **Serilog.Sinks.SentrySDK.AspNetCore.6**.

## Install

```bash
dotnet add package Serilog.Sinks.SentrySDK.10
dotnet add package Serilog.Sinks.SentrySDK.AspNetCore.10
```

These package IDs (**Serilog.Sinks.SentrySDK.6** and **Serilog.Sinks.SentrySDK.AspNetCore.6**) ship **net6.0-only** assemblies (`lib/net6.0/`). They are the same sink and ASP.NET Core integration as the main packages, but published under alternate ids for projects that stay on .NET 6.

For **net6.0 and net10.0** in one package, use **Serilog.Sinks.SentrySDK** and **Serilog.Sinks.SentrySDK.AspNetCore** instead.

## Install

```bash
dotnet add package Serilog.Sinks.SentrySDK.6
dotnet add package Serilog.Sinks.SentrySDK.AspNetCore.6
```
